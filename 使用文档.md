# AI 操作任务使用文档

本文档说明如何在 `TaskPilot` 中通过 AI 聊天操作任务。`TaskPilot` 是当前前端产品名称，对应仓库与后端历史名称 `AI-agent-TODO`。AI 聊天不仅可以普通对话，也可以直接执行任务相关功能。

## 前置条件

1. 启动后端服务。

```bash
cd backend
./.venv/bin/uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

2. 启动前端服务。

```bash
cd frontend
npm run dev
```

3. 打开前端页面。

```text
http://127.0.0.1:5173
```

4. 注册或登录后端账号。

5. 在设置的 AI 配置中确认模型和 API Key 可用。

本地演示模式也能用 AI 聊天操作本地任务，但数据只保存在浏览器本地。登录后端账号后，操作会写入后端数据库。

## 打开 AI 聊天

登录后，在页面右下角点击 AI 聊天按钮，打开聊天浮窗。

如果输入的是任务操作指令，系统会直接执行任务操作。如果输入不是任务操作指令，系统会按普通 AI 聊天处理。

## 追问模式

聊天输入框底部有一个“追问模式”开关，位置在文件按钮右侧。

关闭时，AI 会尽量直接执行识别到的任务操作。

开启时，如果你的要求不够清楚，AI 会先追问你补充信息，而不是直接使用默认值或冒险执行。

适合开启追问模式的情况：

- 创建任务时只说了很短的内容。
- 修改任务时没有说清楚要改哪个任务。
- 删除任务时任务名称太模糊。
- 完成或恢复任务时可能匹配到多个任务。

示例：

```text
创建任务：弄一下
```

追问模式开启时，AI 会提示你补充任务内容、截止时间、优先级或分类。

## 支持的任务功能

当前 AI 聊天支持以下任务操作：

- 查看帮助教程
- 创建任务
- 查询任务列表
- 查看任务详情
- 修改任务
- 删除任务
- 标记任务完成
- 恢复任务为待办

## 查看帮助教程

如果忘记怎么操作，可以直接在 AI 聊天里输入：

```text
help
```

```text
帮助
```

```text
怎么用
```

```text
你能做什么
```

```text
任务帮助
```

AI 会返回当前支持的任务操作和示例。

## 创建任务

示例：

```text
创建任务,test
```

```text
新增任务：明天下午三点完成软件工程报告
```

```text
帮我记下周五交作业
```

```text
把准备答辩 PPT 加入待办
```

创建成功后，AI 会回复任务标题、优先级、分类和截止时间。

```text
查询任务
```

```text

```

查看待办任务：

```text
查看待办任务
```

查看已完成任务：

```text
查看已完成任务
```

查看高优先级任务：

```text
列出高优先级任务
```

按分类查询：

```text

```

```text
查询任务 报告
```

查询结果会显示任务标题、状态、优先级、分类和截止时间。

## 查看任务详情

示例：

```text
查看任务 test
```

```text
任务详情 软件工程报告
```

如果只匹配到一个任务，AI 会显示该任务的详细信息。

如果匹配到多个任务，AI 会列出候选任务，并要求你说得更具体一些，避免误操作。

## 修改任务

修改标题：

```text
修改任务 test 为 完成软件工程测试
```

修改优先级：

```text
修改任务 软件工程报告 优先级为高
```

修改分类：

```text
修改任务 软件工程报告 分类为学习
```

修改状态：

```text
修改任务 软件工程报告 为已完成
```

修改截止时间：

```text
修改任务 软件工程报告 截止时间为2026-06-20 15:00
```

修改描述：

```text
修改任务 软件工程报告 描述为整理需求分析和接口设计
```

说明：后端当前只支持待办和已完成两种状态。如果说“进行中”，系统不会强行写入后端不存在的状态。

## 删除任务

示例：

```text
删除任务 test
```

```text
删掉软件工程报告
```

```text
把准备答辩 PPT 从待办里删除
```

为了避免误删，如果匹配到多个任务，AI 不会直接删除，而是提示你进一步明确任务名称。

## 完成和恢复任务

标记完成：

```text

```

```text

```

恢复为待办：

```text

```

```text

```

## AI 如何执行这些操作

登录后端账号时，任务操作会复用项目现有 API：

```text
POST /api/ai/create-task
GET /api/tasks
GET /api/tasks/{id}
PUT /api/tasks/{id}
DELETE /api/tasks/{id}
PATCH /api/tasks/{id}/status
```

其中创建任务会使用 AI 解析自然语言。查询、查看、修改、删除和状态切换会基于当前任务列表匹配目标任务。

## 匹配规则

AI 会根据任务标题、描述、分类和标签匹配任务。

如果匹配到 0 个任务：

```text
没有找到匹配“xxx”的任务。
```

如果匹配到多个任务：

```text
找到多个匹配“xxx”的任务，请说得更具体一些。
```

这时可以输入更完整的任务标题，或先用“查询任务 xxx”查看候选。

## 普通聊天

如果输入内容不是任务操作指令，系统会继续按普通 AI 聊天处理。

例如：

```text
帮我分析今天应该先做哪个任务
```

这种输入不会直接修改任务数据。

## 推荐写法

推荐把动作和任务名称说清楚：

```text
创建任务：明天下午 3 点完成软件工程实验报告，优先级高，分类项目
```

```text
修改任务 软件工程实验报告 优先级为高
```

```text
完成任务 软件工程实验报告
```

不推荐：

```text
弄一下
```

```text
改了吧
```

## 排查问题

如果 AI 操作任务失败，可以检查：

1. 是否已经登录后端账号。
2. 后端是否运行在 `http://127.0.0.1:8000`。
3. 前端 API 地址是否是 `http://127.0.0.1:8000/api`。
4. 设置页中的模型和 API Key 是否通过“测试连接”。
5. 任务名称是否太模糊，导致匹配不到或匹配到多个任务。
6. 浏览器控制台或页面提示中是否出现网络错误、鉴权错误或模型错误。
