# AI-agent-TODO 网页重设计需求整理

本文用于将当前 AI-agent-TODO 前端交给 Stitch 重新设计。目标不是重新定义产品功能，而是在保留现有功能与信息架构的基础上，重做更统一、更现代、更适合课程项目展示的网页端视觉与交互。

## 1. 产品定位

AI-agent-TODO 是一个带 AI 辅助能力的智能任务管理网页应用。用户可以创建、筛选、编辑、删除任务，并通过 AI 获得任务优先级、分类、截止安排和执行建议。

产品气质建议：

- 面向个人学习、课程项目、开发任务管理和日常待办管理。
- 更像一个工作台型 SaaS 工具，而不是营销官网。
- 界面应安静、清晰、可靠，突出任务处理效率。
- AI 能力应被设计为自然融入任务流，而不是过度装饰。

## 2. 当前主要导航

左侧导航目前保留以下页面：

1. Dashboard
2. 今日任务
3. 全部任务
4. 任务看板
5. 日历
6. 数据统计
7. 设置

说明：

- 原来的独立“AI 推荐”导航入口已移除。
- Dashboard 内仍需保留“AI 智能建议”板块。
- AI 相关能力主要出现在 Dashboard、任务详情、创建任务流程、设置页和右下角 AI 聊天浮窗中。

## 3. 全局布局需求

### 桌面端

当前是典型管理后台结构：

- 左侧固定侧边栏：品牌、主导航。
- 顶部工具栏：全局搜索、新建任务、通知、主题切换、API 状态、用户入口、退出。
- 主内容区：根据页面切换不同工作区。
- 右下角悬浮 AI 聊天窗口。

重设计建议：

- 保持左侧导航 + 顶部工具栏 + 主内容区的信息架构。
- 页面应更紧凑，减少大面积空白。
- 卡片不宜过多嵌套，避免“卡片套卡片”的厚重感。
- 搜索和新建任务是高频操作，应保持明显但不抢占主内容。
- 全局 API 状态可以保留，但视觉上要弱化为状态 chip。

### 移动端

当前移动端有底部导航：

- Dashboard
- 全部任务
- 新建
- 数据统计
- 更多页面入口

重设计建议：

- 底部导航保留，优先保证核心操作可达。
- 新建任务按钮应作为中间主操作。
- 任务列表改为卡片式，避免桌面表格在小屏上横向滚动。

## 5. 核心数据字段

任务字段：

- id
- title：任务标题
- description：任务描述
- status：待办 / 进行中 / 已完成
- priority：高 / 中 / 低
- category：分类
- dueDate：截止日期
- dueTime：截止时间
- createdAt：创建日期
- completedAt：完成日期
- tags：标签
- aiReason：AI 推荐原因
- estimatedTime：AI 预计完成时间
- aiCategory：AI 分类
- isAiCreated：是否 AI 创建
- confidence：AI 解析置信度

状态与视觉建议：

- 高优先级：需要明显但克制。
- 逾期：比高优先级更紧急。
- 已完成：弱化、可加删除线或透明度。
- AI 生成 / AI 推荐：使用小标签，而不是大面积渐变。

## 6. 关键用户流程

### 流程 1：演示登录

1. 打开登录页。
2. 点击“使用演示账号（本地）”。
3. 进入 Dashboard。

### 流程 2：创建任务

1. 点击顶部“新建任务”。
2. 填写任务标题、描述、优先级、分类、截止时间。
3. 点击创建任务。
4. 任务出现在 Dashboard、全部任务、看板、日历和统计中。

### 流程 3：处理今日任务

1. 进入 Dashboard 或今日任务页。
2. 查看 AI 智能建议和今日任务。
3. 点击任务查看详情。
4. 标记完成。
5. 完成率和统计同步更新。

### 流程 4：筛选和管理任务

1. 进入全部任务。
2. 搜索或筛选状态、优先级、分类。
3. 打开详情、编辑、删除或切换完成状态。

### 流程 5：使用看板

1. 进入任务看板。
2. 按状态查看任务列。
3. 通过筛选缩小任务范围。
4. 点击卡片打开详情。

### 流程 6：查看日程

1. 进入日历。
2. 切换 7 天、14 天、30 天、今日 24 小时或逾期。
3. 点击日历任务查看详情。

### 流程 7：查看统计

1. 进入数据统计。
2. 查看任务总量、完成率、逾期、AI 创建数量。
3. 查看趋势、分类分布和优先级分布。

### 流程 8：AI 聊天

1. 点击右下角 AI 聊天入口。
2. 输入问题或选择附件 metadata。
3. 选择模型。
4. 发送消息。
5. 查看对话记录或新建对话。

## 7. 当前设计问题与重设计重点

当前问题：

- 页面整体已经可用，但视觉层级还可以更统一。
- 部分区域卡片感较重，信息密度与留白需要重新平衡。
- AI 能力分散在多个位置，需要更自然的设计语言串联。
- 表格、看板、日历、统计之间的视觉系统要更一致。
- 移动端需要更明确的底部导航和任务卡片体验。
- AI 聊天浮窗已经调整过多次，建议重设计时整体重新梳理，而不是局部微调。

重设计目标：

- 更现代、更干净、更适合展示的工作台界面。
- 保留管理后台的清晰结构，但避免模板化后台感。
- 让 Dashboard 成为真正的任务决策中心。
- 让 AI 能力看起来可信、辅助、低干扰。
- 所有页面共享一致的按钮、标签、卡片、状态、表单、空状态规范。
- 响应式设计必须覆盖桌面和移动端。

## 8. 视觉方向建议

推荐方向：

- 背景：浅色为主，使用低饱和灰白工作台背景。
- 主色：蓝色或蓝绿色均可，体现效率和智能。
- 辅助色：
  - 绿色：完成
  - 红色：逾期 / 删除
  - 橙色：高优先级或临近截止
  - 紫色：AI 相关标签，但不要大面积使用
- 字体：系统字体，中文优先清晰可读。
- 圆角：中等圆角，避免过度圆润。
- 阴影：轻量，少用厚重阴影。
- 图标：线性图标风格，和当前 lucide 风格一致。

避免：

- 大面积紫蓝渐变。
- 过度营销化 hero。
- 装饰性图形太多。
- 复杂拟物风格。
- 低信息密度的大卡片堆叠。
