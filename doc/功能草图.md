## 5. AI 自然语言创建流程

```mermaid
sequenceDiagram
  participant User as 用户
  participant TasksPage as 任务页
  participant Backend as FastAPI 后端
  participant AIService as AI 服务
  participant OpenAI as OpenAI API
  participant DB as 数据库

  User->>TasksPage: 输入自然语言任务
  User->>TasksPage: 点击解析
  TasksPage->>Backend: POST /api/ai/parse-task
  Backend->>AIService: 校验用户设置和输入文本
  AIService->>OpenAI: 使用用户 API Key 请求解析
  OpenAI-->>AIService: 返回结构化任务字段
  AIService-->>Backend: 返回解析结果
  Backend-->>TasksPage: 返回 AI 草稿
  TasksPage-->>User: 展示标题 优先级 分类 截止时间
  User->>TasksPage: 修正草稿并确认创建
  TasksPage->>Backend: POST /api/ai/create-task
  Backend->>DB: 创建任务并记录 AI 调用
  Backend-->>TasksPage: 返回新任务
  TasksPage-->>User: 刷新任务列表和统计
```

## 7. 核心接口关系

| 页面 | 主要接口 | 用途 |
| --- | --- | --- |
| `/login` | `/api/auth/register`、`/api/auth/login` | 用户注册和登录 |
| `/tasks` | `/api/tasks`、`/api/tasks/{task_id}`、`/api/tasks/{task_id}/status` | 任务增删改查和状态切换 |
| `/tasks` | `/api/ai/parse-task`、`/api/ai/create-task` | AI 解析与 AI 创建任务 |
| `/stats` | `/api/stats/overview`、`/api/stats/category`、`/api/stats/priority`、`/api/stats/trend` | 统计看板数据 |
| `/settings` | `/api/settings`、`/api/settings/test-openai-key` | BYOK 配置和 Key 测试 |

## 8. 展示闭环

```mermaid
flowchart TD
  A["注册或登录"] --> B["配置 OpenAI API Key"]
  B --> C["输入自然语言任务"]
  C --> D["AI 解析任务草稿"]
  D --> E["确认创建任务"]
  E --> F["在任务列表管理任务"]
  F --> G["切换完成状态"]
  G --> H["查看统计看板"]
  H --> I["回到任务页继续管理"]
```

该闭环能够在产品展示中快速说明项目的核心价值：用户完成登录后，可以配置 AI 能力，通过自然语言创建任务，在任务工作台持续管理，并通过统计看板获得执行反馈。
