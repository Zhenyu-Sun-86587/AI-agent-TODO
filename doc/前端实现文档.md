# TaskPilot 前端实现文档

更新时间：2026-06-19

## 1. 使用技术

TaskPilot 前端使用 React、TypeScript 和 Vite 实现，代码位于 `frontend/` 目录。

主要技术栈：

- React：负责页面组件、状态组织和交互逻辑。
- TypeScript：为任务、用户、设置、AI 聊天、接口返回值等核心数据结构提供类型约束。
- Vite：负责本地开发服务、构建和前端资源打包。
- CSS：采用全局样式文件组织设计 token、基础样式、布局样式、工具类和业务页面样式。
- lucide-react：提供导航、按钮、状态和表单中的线性图标。
- Fetch API：通过封装后的 API 模块请求后端接口。
- localStorage：保存登录会话、主题、AI 聊天会话、当前模型和部分本地演示数据。
- Playwright：用于端到端测试和视觉审计脚本。

前端开发命令：

```bash
cd frontend
npm install
npm run dev
npm run build
npm run test:e2e
```

开发环境接口地址通过 `VITE_API_BASE_URL` 配置。当前约定如果环境变量已经包含 `/api`，前端 API 模块内部只调用 `/auth/login`、`/tasks`、`/stats/overview` 等业务路径。

当前用户可见产品名统一为 **TaskPilot**。`AI-agent-TODO` 保留为仓库名、后端应用名、历史文档名和本地存储 key 前缀。

## 2. 前端设计内容

### 2.1 整体信息架构

TaskPilot 是一个带 AI 辅助能力的智能任务管理网页应用。用户可以创建、筛选、编辑、删除任务，并通过 AI 获得任务优先级、分类、截止安排和执行建议。

主应用采用工作台结构：

- 左侧侧边栏：品牌、主导航、用户入口、设置入口、登录状态和退出登录。
- 顶部工具栏：全局搜索、新建任务、通知、主题切换和移动端更多入口。
- 主内容区：根据当前路由展示仪表盘、任务中心、日历、智能助手或设置内容。
- 右下角：常驻 AI 聊天浮窗。
- 移动端：使用底部导航承载高频页面和新建任务入口。

左侧主导航包含：

1. 仪表盘
2. 任务中心
3. 日历
4. 智能助手

设置页通过侧边栏用户区的齿轮按钮进入；个人资料和退出登录通过用户菜单进入。

### 2.2 登录与认证体验

登录页使用沉浸式封面设计：

- 背景图：`frontend/public/background.png`
- 标题框资源：`frontend/public/kuang.png`
- 产品标题：TaskPilot
- 标语：把想法变成可执行任务
- 入口提示：点击页面任意位置，登录并进入任务工作台

登录/注册卡片包含：

- 登录和注册 tabs
- 邮箱、密码字段
- 注册时显示用户名和确认密码字段
- 登录并进入、注册并进入、使用演示账号体验
- API 状态提示和错误提示

登录成功和退出登录使用短促衔接动画，只做轻量位移和透明度变化，避免整页长过场或白色空档。

### 2.3 工作台页面

仪表盘是进入系统后的默认首页，强调“今天该做什么”。主要内容包括：

- 欢迎区和今日状态提示
- 今日任务、已完成、逾期任务、AI 推荐优先等概览指标
- AI 智能建议列表
- 今日任务列表

任务中心负责主要任务管理流程：

- 全局搜索联动
- 状态、优先级、分类筛选
- 排序和任务列表展示
- 新建、编辑、删除、完成/取消完成
- 任务详情抽屉
- AI 字段推荐和 AI 生成相关信息展示

日历页面负责按日期查看任务，帮助用户从时间维度检查截止任务和安排。

智能助手页面集中展示 AI 相关工作区能力，包括任务建议、字段推荐和 AI 日志等内容。

设置相关界面包括：

- 个人资料弹窗
- 设置弹窗
- 主题切换
- OpenAI/DeepSeek 模型和 API Key 配置
- API 连通性测试和联调信息

### 2.4 AI 聊天浮窗

AI 聊天浮窗位于右下角，作为主工作台之外的常驻辅助入口。保留能力包括：

- 聊天记录
- 创建新对话
- 会话切换和删除
- 模型切换
- 文件选择和提交 metadata

AI 聊天数据使用 localStorage 保存：

- `ai-agent-todo.ai-chat.conversations`
- `ai-agent-todo.ai-chat.activeConversationId`
- `ai-agent-todo.ai-chat.selectedModelId`

当前模型分组包括 GPT 和 DeepSeek。真实接口接入点集中在 `aiClient.ts`，后续可以按模型 provider 路由到后端 OpenAI 或 DeepSeek 代理。API Key 不应放在前端组件或前端配置中，应由后端代理持有和调用。

### 2.5 视觉与交互原则

当前前端视觉方向是克制的任务工作台，而不是营销官网：

- 浅色为默认工作台基调，支持暗色主题。
- 主色用于关键操作，绿色表示完成，红色表示删除/逾期，橙色表示高优先级或临近截止，紫色只作为 AI 标签点缀。
- 卡片和弹窗保持清晰层级，避免过多装饰。
- 图标统一使用线性风格。
- 表单、任务卡片、抽屉和弹窗需要保证移动端可读、可操作。
- 登录/退出、弹窗关闭、移动端更多菜单等交互使用短动画增强衔接，不做影响效率的长动画。

## 3. 相关代码结构

### 3.1 顶层入口

```text
frontend/
├── index.html
├── package.json
├── vite.config.ts
├── playwright.config.ts
└── src/
    ├── main.tsx
    └── App.tsx
```

- `frontend/index.html`：HTML 入口，浏览器标题为 TaskPilot。
- `frontend/src/main.tsx`：React 挂载入口，加载全局样式。
- `frontend/src/App.tsx`：应用主容器，管理登录态、主题、页面路由、任务数据、设置弹窗、任务弹窗、AI 浮窗和 toast。

### 3.2 路由与页面

```text
frontend/src/app/router/routes.ts
frontend/src/pages/
├── AuthPage.tsx
├── DashboardPage.tsx
├── TasksPage.tsx
├── CalendarPage.tsx
├── SettingsPage.tsx
├── PageHeading.tsx
└── workspacePages.tsx
```

- `routes.ts`：维护页面 key 和浏览器路径映射。
- `AuthPage.tsx`：登录/注册封面页。
- `workspacePages.tsx`：根据当前页面 key 渲染对应工作区页面。
- `DashboardPage.tsx`：仪表盘页面。
- `TasksPage.tsx`：任务中心页面。
- `CalendarPage.tsx`：日历页面。
- `SettingsPage.tsx`：设置页面入口。

### 3.3 布局组件

```text
frontend/src/components/layout/
├── Layout.tsx
├── Sidebar.tsx
├── Header.tsx
├── MobileNav.tsx
├── NavItem.tsx
└── types.ts
```

- `Layout.tsx`：主应用布局容器。
- `Sidebar.tsx`：桌面端侧边栏和用户菜单。
- `Header.tsx`：顶部工具栏。
- `MobileNav.tsx`：移动端底部导航。
- `NavItem.tsx`：导航项组件。
- `types.ts`：布局相关类型。

### 3.4 API 模块

```text
frontend/src/api/
├── client.ts
├── auth.ts
├── tasks.ts
├── stats.ts
├── settings.ts
├── ai.ts
├── mappers.ts
├── errors.ts
└── types.ts
```

- `client.ts`：封装 API 基础地址、请求头、错误判断和响应 envelope 解析。
- `auth.ts`：登录、注册、演示登录、退出登录、获取/更新当前用户。
- `tasks.ts`：任务 CRUD 和任务列表查询。
- `stats.ts`：统计数据接口。
- `settings.ts`：AI 设置保存、读取和测试。
- `ai.ts`：AI 推荐、日志等接口。
- `mappers.ts`：后端任务数据到前端任务模型的转换。
- `errors.ts`：API 错误类型和错误信息格式化。
- `types.ts`：后端接口类型定义。

### 3.5 功能模块

```text
frontend/src/features/
├── auth/
├── dashboard/
├── tasks/
├── settings/
├── ai/
└── ai-chat/
```

认证模块：

- `features/auth/hooks/useAuth.ts`：登录、注册、演示登录、退出登录、会话保存和认证错误处理。
- `features/auth/types.ts`：会话类型。
- `features/auth/utils/validation.ts`：邮箱等校验逻辑。

任务模块：

- `features/tasks/hooks/useTasks.ts`：任务加载、本地演示数据、远程任务同步、创建、编辑、删除、完成、详情加载。
- `features/tasks/components/`：任务列表、卡片、看板、过滤器、编辑器、详情抽屉和弹窗。
- `features/tasks/services/mockData.ts`：本地演示任务。
- `features/tasks/utils/generation.ts`：任务生成辅助逻辑。
- `features/tasks/types.ts`：任务类型。

仪表盘模块：

- `features/dashboard/hooks/useDashboardData.ts`：仪表盘统计和推荐任务计算。
- `features/dashboard/components/`：概览卡片和推荐任务组件。

设置模块：

- `features/settings/hooks/useSettings.ts`：主题、个人资料、AI 设置读取和保存。
- `features/settings/components/`：模型设置、个人资料设置。
- `features/settings/types.ts`：设置类型。

AI 工作区：

- `features/ai/components/AiWorkspace.tsx`：智能助手页面。

AI 聊天浮窗：

- `components/ai-chat/FloatingChat.tsx`：浮窗容器、会话状态、localStorage 同步、发送流程。
- `components/ai-chat/ChatThread.tsx`：消息列表、空会话欢迎语、自动滚动。
- `components/ai-chat/ChatComposer.tsx`：输入框、文件 metadata 选择、模型选择、发送。
- `components/ai-chat/ConversationMenu.tsx`：会话列表、切换、删除。
- `components/ai-chat/types.ts`：Conversation、Message、Attachment、Model 类型。
- `components/ai-chat/storage.ts`：localStorage key、读取、写入、数据归一化。
- `components/ai-chat/models.ts`：模型配置与 GPT / DeepSeek 分组。
- `components/ai-chat/aiClient.ts`：真实接口接入点。
- `components/ai-chat/FloatingChat.css`：浮窗专属样式。

## 4. 后续维护建议

- 新增页面或调整导航后，同步更新 `App.tsx`、`routes.ts`、布局组件和本文档。
- 修改产品名、默认演示账号、登录页资源时，同步检查 `frontend/index.html`、`AuthPage.tsx`、`Sidebar.tsx`、`frontend/README.md` 和本文档。
- 新增后端接口时，优先在 `frontend/src/api/` 中增加类型和请求封装，再由 feature hook 调用。
- AI 聊天接入真实上传或流式响应时，优先扩展 `components/ai-chat/aiClient.ts`，避免把服务商密钥放入前端。
