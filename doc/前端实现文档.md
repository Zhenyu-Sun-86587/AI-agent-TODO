## 1. 文档说明

本文档描述当前 `frontend/` 目录中的真实前端实现。

命名约定：

- 仓库名、部分历史存储 key 和后端应用名：`AI-agent-TODO`

## 2. 使用技术

当前前端使用：

- React 19
- TypeScript
- Vite
- Fetch API
- lucide-react
- localStorage
- Playwright

当前项目不依赖：

- Axios
- React Router
- Zustand / Redux

页面切换使用应用内自定义路由状态与 `history` 同步，接口访问使用 [frontend/src/api/client.ts](/Users/fiwoouo/Desktop/软工实验/AI-agent-TODO/frontend/src/api/client.ts:1)。

## 3. 启动与构建

```bash
cd frontend
npm install
npm run dev
npm run build
npm run test:e2e
```

开发环境 API 地址：

```text
VITE_API_BASE_URL=http://127.0.0.1:8000/api
```

## 4. 产品结构

TaskPilot 是一个以任务工作台为核心的 Web 应用，主要页面和入口包括：

- 登录 / 注册页
- 仪表盘
- 任务中心
- 日历页
- 智能助手工作区
- 右下角 AI 聊天浮窗
- 个人资料与设置弹窗

说明：

- 当前没有单独的 `SettingsPage.tsx`
- 设置以弹窗和设置组件形式存在于工作台中

## 8. API 模块

```text
frontend/src/api/
├── client.ts
├── auth.ts
├── tasks.ts
├── stats.ts
├── settings.ts
├── ai.ts
├── mappers.ts
├── errors.ts
└── types.ts
```

说明：

- `client.ts`：Fetch 封装、统一请求头和 envelope 解析
- `auth.ts`：注册、登录、演示登录、登出、`/users/me`
- `tasks.ts`：任务 CRUD、分页查询、分类聚合、AI 创建任务
- `stats.ts`：overview/category/priority/trend
- `settings.ts`：设置读取、保存、测试 Key
- `ai.ts`：AI 推荐、AI 日志、AI 聊天
- `types.ts`：后端响应类型
- `mappers.ts`：后端字段到前端任务展示模型的映射

## 13. 维护建议

- 调整页面映射时，同步检查 `App.tsx`、`routes.ts`、`workspacePages.tsx`
- 调整 API 时，同步更新 `src/api/*.ts`、`src/api/types.ts` 和本文件
- 调整样式分层时，以 `frontend/src/styles/README.md` 为准更新本文档
- 调整用户可见产品名或 demo 账号时，同步检查登录页、侧边栏、README 和测试
