# 项目方案书：AI-agent-TODO 智能任务管理系统

> 历史设计文档：本文档用于保留课程立项与早期方案。当前实现请优先参考 `README.md`、`doc/前后端API接口文档.md`、`doc/前端实现文档.md`、`doc/后端实现文档.md` 与实际源码。

## 一、项目名称

**AI-agent-TODO 智能任务管理系统**

## 二、项目背景

传统 TODO 软件主要依赖用户手动创建任务、设置时间和分类，操作较繁琐，且缺少智能分析能力。本项目拟开发一款结合 AI Agent 的任务管理系统，通过自然语言解析、智能分类、优先级推荐等功能，提高用户任务管理效率。

## 三、项目目标

本项目计划实现一个以 Web 端为核心的智能 TODO 管理系统，支持用户注册登录、任务创建、编辑、删除、完成状态管理、任务分类、任务统计等功能。

同时，系统将引入 OpenAI API，采用 BYOK 模式，由用户自行配置 API Key，实现 AI 辅助任务管理。

后续可扩展为移动端和桌面端应用，实现多端统一使用。

## 四、系统主要功能

### 1. 用户管理

用户可注册、登录和管理个人信息。系统根据用户身份隔离任务数据，保证不同用户之间的数据独立。

### 2. 任务管理

用户可创建、查看、修改、删除任务，并可设置任务截止时间、优先级、分类和完成状态。

### 3. AI 自然语言解析

用户可输入自然语言任务，例如：

> 明天下午三点完成软件工程报告，很重要

系统通过 AI 自动解析为：

```
{
  "title": "完成软件工程报告",
  "due_time": "明天下午三点",
  "priority": "high",
  "category": "学习"
}
```

### 4. 智能分类与优先级推荐

系统根据任务内容自动推荐任务分类和优先级，减少用户手动设置成本。

### 5. 任务统计分析

系统提供任务完成率、未完成任务数量、不同分类任务数量等统计信息，帮助用户了解任务完成情况。

### 6. BYOK 设置

用户可在系统设置中填写自己的 OpenAI API Key，系统使用该 Key 调用 AI 能力，降低项目部署成本。

## 五、技术方案

### 前端

采用 **React + Vite** 构建 Web 前端页面，负责用户交互、任务展示和 AI 输入界面。

### 后端

采用 **FastAPI** 构建后端服务，提供用户管理、任务管理、AI 调用和数据统计接口。

### 数据库

采用 **SQLite / PostgreSQL** 存储用户信息、任务信息、AI 调用记录和系统配置。

### AI 能力

基于 **OpenAI API** 实现自然语言任务解析、任务分类、优先级推荐和任务总结等功能。

### 扩展方向

后续可基于同一后端 API 扩展：

- 移动端 App
- 桌面端应用
- 多模型支持
- 多人协作任务管理

## 六、系统架构

```
Web 前端 React
      ↓
FastAPI 后端服务
      ↓
数据库 SQLite / PostgreSQL
      ↓
OpenAI API（BYOK）
```

## 七、项目创新点

1. 将 AI Agent 引入 TODO 任务管理场景。
2. 支持自然语言创建任务，降低用户输入成本。
3. 支持 BYOK 模式，用户可自行配置 OpenAI API Key。
4. 支持任务智能分类、优先级推荐和统计分析。
5. 系统架构具备扩展性，可拓展移动端和桌面端。
